# 🏸 基于 MediaPipe 与 Skeleton Transformer 的羽毛球击球动作识别

## 📖 项目简介
本项目为《计算机视觉》课程期末实验作业。项目旨在将**视频动作识别任务**转化为**骨架时间序列分类任务**。相较于直接使用庞大的 3D-CNN 或 Video Transformer 处理原始视频像素，本项目利用 MediaPipe Pose 提取人体骨架关键点，并使用轻量级的 **Skeleton Transformer** 进行序列建模。该方法计算量低、可解释性强，能够高效完成羽毛球 6 大类击球动作的分类。

## 💾 存储路径与盘符说明（重要）
由于本地 C 盘空间不足，为了保护系统盘空间，本项目的所有核心数据集、权重文件及测试视频已**统一存放在 Windows 的 D 盘**。在 WSL (Ubuntu) 子系统中运行时，通过 Linux 挂载路径 `/mnt/d/computervisionlab/` 进行数据读写。

具体文件布局如下：
D:\computervisionlab\

├── X_train.npy                # 预处理生成的训练集特征
├── y_train.npy                # 预处理生成的训练集标签
├── X_test.npy                 # 预处理生成的测试集特征
├── y_test.npy                 # 预处理生成的测试集标签

└── demo_video.mp4             # 用于单样本推理测试的 Demo 视频

## 📊 数据集类别
本项目使用 Kaggle 提供的 `badminton_storke_video` 数据集，共包含 6 类羽毛球动作片段：
1. `forehand_drive` (正手平抽)
2. `forehand_lift` (正手挑球)
3. `forehand_net_shot` (正手网前球)
4. `forehand_clear` (正手高远球)
5. `backhand_drive` (反手平抽)
6. `backhand_net_shot` (反手网前球)

## 🧠 模型架构 (Skeleton Transformer)
模型采用轻量级 Transformer Encoder 结构，核心参数与数据流向如下：
* **输入特征**: `[Batch_Size, 30, 132]` (将视频重采样为统一 30 帧，每帧提取 33 个关键点 × 4 个维度 `x, y, z, visibility`)
* **Linear Embedding**: 132 维原始物理坐标通过全连接层映射至 `d_model = 128` 的高级特征空间。
* **Positional Encoding**: 标准正弦/余弦绝对位置编码（Sinusoidal Position Embedding），为无序的 Attention 机制赋予时序感知能力。
* **Transformer Encoder**: 包含 2 层 Encoder Layer (`num_layers = 2`)，使用 4 头注意力机制 (`nhead = 4`)，前馈时序维度 `dim_feedforward = 256`，`dropout = 0.1`。
* **分类头 (Classifier)**: 对时间维度进行 Mean Pooling（平均池化）后接入 MLP 分类器，最终输出 6 类动作的 Logits。

pip install torch numpy opencv-python mediapipe scikit-learn matplotlib seaborn pandas

## 🚀 快速开始

### 1. 数据预处理
运行预处理脚本，遍历 D 盘原始视频，使用 MediaPipe 提取关键点，将视频重采样为统一的 30 帧序列，按 `test_size=0.2` 比例划分训练集与测试集，并执行分层抽样 (`stratify`)，最终直接保存回 D 盘：
python preprocess.py
*主要生成文件*：`X_train.npy`, `X_test.npy`, `y_train.npy`, `y_test.npy`

### 2. 模型训练与评估
从 D 盘路径加载切分好的 `.npy` 数据进行训练。训练完成后，脚本会自动打印最佳模型的混淆矩阵和分类报告，并自动在当前目录下生成两张高清可视化 JPG 评估图表：
python train.py
*主要生成文件*：
* `transformer_best.pth` (最优模型权重，保存于 D 盘)
* `evaluation_results.jpg` (实时动态混淆矩阵与分类指标热力图，保存于项目本地)
* `training_curves.jpg` (实时训练 Loss 曲线与 Train/Test Accuracy 波动曲线，保存于项目本地)

### 3. 单样本推理 (Inference)
选取任一测试视频命名为 `demo_video.mp4` 并放置于 `D:\computervisionlab\` 目录下。运行推理脚本，系统会自动读取视频、提取骨架、加载最优权重、经过 Softmax 映射后输出预测类别及置信度：
python inference.py

## 📈 实验结果分析
在没有进行任何数据增强的前提下，仅迭代训练 30 个 Epoch，模型在测试集上稳定取得了约 **42% ~ 43%** 的最高准确率。鉴于 6 分类任务的随机猜测概率仅为 16.6%，且 Transformer 模型在小样本（训练集仅 668 个序列）下极易发生过拟合，当前网络已经展现出了良好的时序动作模式抓取能力。

通过导出的 `evaluation_results.jpg` 可以直观地看到，模型对 `backhand_net_shot` (反手网前球) 的精确率最高（达 0.68），而容易将 `forehand_clear` (正手高远球) 与 `forehand_drive` (正手平抽) 产生混淆，这与真实物理世界中两者引拍动作的相似性具有高度的技术吻合度。

## 💡 总结与反思
本项目成功闭环了“视频读取 -> 骨架提取与轻量化 -> 时序 Transformer 建模 -> 指标可视化与单样本推理”的全工作流。通过将物理坐标无缝转入 D 盘存储，成功克服了 C 盘空间受限的硬件约束。

**未来改进方向**：
1. **数据增强**：对 132 维骨架序列在时序上进行轻微缩放、空间旋转或叠加热力图高斯噪声，以抑制过拟合。
2. **多模态特征融合**：除关键点坐标外，进一步计算关节之间的相对角度、速度与加速度作为补充 token，以进一步提升模型的分类精度。
